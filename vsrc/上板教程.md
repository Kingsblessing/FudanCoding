# 上板教程



## 备注



Vivado 里面对代码的要求跟 Verilator 不太一样。并且我们实际上很难说谁更严格。你需要保证你的代码使用了 Vivado 和 Verilator 支持的语法和特性的交集。

否则，你可以使用 ` `ifdef VERILATOR` 宏来为 Vivado 和 Verilator 分别写不同的代码。

在 Vivado 当中 `initial` 就不应当被使用。且 `include` 语句也不应当被使用，因此我们范例的 `include` 语句都是被 `ifdef VERILATOR` 包裹的。

在 Vivado 中，定义直接用等号是不行的。例如 `logic [6:0]  opcode = instr[6:0];` 会导致 `opcode` 一直为 `XX` 状态。请改成 `logic [6:0] opcode; assign opcode = instr[6:0];`

## 准备



1. 上板之前，首先要保证你的代码能够在 Verilator 中运行 test-lab3。
2. 安装 Vivado。
3. 你需要一个软件，能从 UART 当中读取输出并显示在屏幕上。你可以使用任意软件来做到这一功能。
   - 去年的推荐是从 Microsoft Store 中下载安装“串口调试助手”软件。
   - 也可以使用 [PuTTY](https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html)。
   - 如果你之前玩过 Arduino，也可以用 Arduino IDE 自带的串口监视器。
4. 如果你的arch-2025 项目位于WSL的路径中，需要将其复制到 Windows 下。注意路径不能含有中文、长度不能太长。同时要注意，复制到 Windows 下后，对代码的改动不会同步回 WSL，你需要手动同步。
5. 测试电脑与开发板的连接：
   1. 打开代码目录里面的 `vivado/test_cpu/project/project_1.xpr`。用数据线连接开发板，将开发板的开关打开。此时开发板应该会亮灯
   2. 左下角 `Open Hardware Manager`-`Open Target`-`Auto Connect`，Vivado应该能够成功连接开发板。如果连接失败，可能是数据线有问题，可以自己换一根线或者联系助教更换
   3. 打开“串口调试助手”软件，选择“端口名”为 COMx ，这里的 x 是一个数字，不同设备不一样。如果有多个 COM 端口，应该选择类型为 USB 设备的。然后点击“打开”
   4. 按下开发板上的 RESET 按键，在串口调试助手中应该能看到输出，证明连接成功

## 导入代码到 Vivado 与运行



> 请确保你完成了上面的 5.1 步骤。

1. 点击 Vivado 的

    

   ```
   File
   ```

   \-

   ```
   Add Sources
   ```

   \-

   ```
   design sources
   ```

   ，点击

    

   ```
   Add directories
   ```

    

   添加你的

    

   ```
   vsrc
   ```

    

   目录。

   - 在下方**不要**选择 Copy sources into project，要选择 Add sources from subdirectories。导入完成后，你应该在 Sources 的树形目录中找到你自己写的 core 等模块。 ![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Hierarchy.png)

2. 点击 Sources 框中的

    

   ```
   IP Sources
   ```

    

   选项卡。如果你使用的是 Vivado 2018 版本，这里应该如下图所示，不需要操作：

   - 如果你用的不是 2018 版本。那么两个 IP 核可能会显示一个上锁的图案。此时需要更新 IP 核版本，请双击直接更新即可，如果有弹窗请点击“确认”对应的按钮 ![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/IP_core.png)

3. `Run Synthesis`。Synthesis 结束后，点击下方的"Messages"界面，检查其中的所有Error, Critical Warning 和 Warning。你应该去除所有自己代码中的警告信息（可以忽略来自助教提供的框架代码中的 Warning）

4. `Run Implementation`。同样检查警告信息，不应该有"The design failed to meet the timing requirements"的警告。

5. `Generate Bitstream`

6. 在串口调试助手中连接开发板，在 Vivado 中 `Program Device`。如果一切正常，你应该能看到test-lab3 的测试程序开始运行，并在串口调试助手中输出。如果没有输出，可以尝试按一下板子中间的 BTNC 按键，发送一个 Reset 信号

7. 程序运行到最后，正常退出即为测试通过。不会有 HIT GOOD TRAP。

8. 如果你写了乘除法器，可将

    

   ```
   vivado/test-cpu/src/ip/bram_0
   ```

    

   中这一行

    

   ```
   <spirit:configurableElementValue spirit:referenceId="PARAM_VALUE.Coe_File"> ./ ./ ./ ./ ./ready-torun/lab3/lab3-test.coe /spirit:configurableElementValue>
   ```

    

   中的 coe 文件替换为

    

   ```
   lab3-extra-test.coe
   ```

   - 或者也可以在图形页面操作：在 IP Sources 右键 `bram_0`，选择 `Re-customize IP`，在弹出的界面中选择`Other Options`，选择 `lab3-extra-test.coe`，之后你需要重新生成所有东西。如果有弹窗，请直接确认 Generate。 ![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Modify_coe.png)

## 仿真



实际上，在上一步的 4 之前，你也可以先在 Vivado 中仿真一下你的代码，看看是否能正确输出。 **如果 Vivado 的仿真都不正常，那么你的代码上板一定会不正常。** 因此建议你在上板之前先在 Vivado 中仿真一下，先把仿真调正常。

你只需要 `Run Simulation`-`Run behavioral simulation` 然后看一下波形图当中你的 pc 是不是正常在走。

![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Simulation.png)

先在左侧 Scope 找到你的包含 pc 的模块，找到 pc 的信号，直接拖到右侧，然后运行仿真。

![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Wave.png)

如果仿真没问题，那么下方 Tcl Console 应该会输出与你在 Verilator 中看到的类似的输出。

![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Simulation_output.png)

## 时序检查



如果无法 implementation 通过，那么很可能是你的代码没有满足时序要求了。你需要打开 Vivado 的 Timing 界面，看看哪里没有满足时序要求了。

![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Timing.png)

如果有弹窗，请点击确认。

查看 Timing 的 Design Timing Summary，应该保证这三个都是正数。

![img](https://github.com/26-Arch/26-Arch/wiki/Vivado/Hold_time.png)
